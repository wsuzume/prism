Stateless Session
Reverse Proxy

Prism は AES-GCM 暗号化付き Double-Submit Cookie によって、
サーバーサイドのセッションストアを必要とせずに安全なセッション管理を実現します。

Features

🔒

Stateless Session

セッションストアが不要。暗号化された Cookie だけでセッションを完結させます。

🛡️

CSRF Protection

Double-Submit Cookie パターンにより CSRF 攻撃を防ぎます。SubmitHeader の検証で正規リクエストを識別します。

🔑

AES-GCM JWT

標準 JWT を独自拡張した AES-GCM JWT により、ペイロードを暗号化・完全性検証します。

🔄

Auto Token Refresh

SecretCookie の有効期限が切れても AccessCookie が有効であれば自動更新します。

⚡

Reverse Proxy

既存のバックエンドの前段に置くだけで動作します。アプリ側の変更は最小限。

📦

Go Package

pkg/ として公開されており、他のプロジェクトから直接インポートして利用できます。

How it works

  Browser                  Prism (Reverse Proxy)              Backend
  ──────                   ─────────────────────              ───────

  GET /page  ─────────────▶  verify SecretCookie  ──────────▶  handler
                             inject secretPayload
             ◀─────────────  set-cookie (if refresh) ◀────────  response

  POST /api  ─────────────▶  verify SecretCookie   ──────────▶  handler
  + SubmitHeader            verify AccessCookie              (sees accessPayload
             ◀─────────────  inject accessPayload ◀────────────   → CSRF verified)
                             update cookies (if changed)

Token Format

SecretCookie
  base64url(AES-GCM(
    base64url(json(JwtHeader)) . base64url(json(JwtClaims{ secretPayload }))
  ))

AccessCookie
  base64url(AES-GCM(
    base64url(json(JwtHeader)) . base64url(json(JwtClaims{ accessPayload }))
  )) . base64url(publicPayload)  ← AAD (tamper-evident, readable by client)

Payload 比較

Payload クライアントから読める 改ざん検知 用途例
secretPayload ✗ 暗号化 ✓ AES-GCM ユーザーID、権限
accessPayload ✗ 暗号化 ✓ AES-GCM セッション情報 (POST/PUT 時のみ送信)
publicPayload ✓ 平文 AAD ✓ AES-GCM AAD 表示用ユーザー名、テーマ設定

Quick Start

# 1. リポジトリのクローン
git clone https://github.com/wsuzume/prism

# 2. サンプルアプリをビルド
make build

# リリースモード（デバッグログを抑制）
MODE=release make build

# 3. サンプルアプリを起動
make dev

Go パッケージとして利用

go get github.com/wsuzume/prism

開発環境のセットアップ

# Docker 開発イメージのビルド
make image

# 開発シェルへ入る（カレントディレクトリを /work にマウント）
make shell

# シェル内でビルド・フォーマット・テスト
make proxy    # プロダクション用リバースプロキシのビルド
make format   # コードのフォーマット
make test     # テストの実行
